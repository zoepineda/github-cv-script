# Career OS

Engineering intelligence from your GitHub contributions. Track merged PRs, filter by category and date, and turn engineering activity into structured career data.

Built with Next.js, Prisma, SQLite, and Tailwind CSS.

## What it does

- Pulls all merged PRs from your GitHub repositories via the GitHub CLI
- Categorizes them as **Frontend**, **Backend**, **Fullstack**, or **Other**
- Tags achievement types: **Feature**, **Fix**, **Refactor**
- Stores everything in a local SQLite database (no server setup required)
- Displays a filterable, searchable dashboard with GitHub Primer dark theme

## Getting started

### Prerequisites

- Node.js 18+
- [GitHub CLI](https://cli.github.com/) (`gh`) authenticated

### Setup

```bash
npm install
npx prisma generate
npx prisma db push
```

### Seed the database

If you have an existing `prs.json` (generated by the legacy Python script):

```bash
npm run db:seed
```

To fetch fresh PR data, use the legacy script first:

```bash
python3 legacy/fetch_prs.py
npm run db:seed
```

### Run

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).


## Database

Uses SQLite via Prisma — just a file, no database server. The `dev.db` file is gitignored and created locally when you run `npx prisma db push`.

### Models

- **Repository** — GitHub repos (owner, name)
- **PR** — Merged pull requests with metadata, categories, and summary text
- **SyncLog** — Ingestion run history

### Schema includes future-ready fields

The PR model has columns for AI-generated content (`impactSummary`, `resumeBullet`, `skillTags`, `impactScore`) that aren't populated yet. These support planned features like impact analysis and resume generation.
